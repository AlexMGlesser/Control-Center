Control Center Project Status

Purpose

Workspace Overview
- Root workspace: D:\Control-Center
- Core desktop app project: D:\Control-Center\control-app
- Design docs folder: D:\Control-Center\design-documents
- App folders currently present: project-app, work-app, calendar-app, drawing-app, news-app, music-app, server-manager-app, movie-app

Current App Runtime Model
- Control Center is now a desktop app runtime using Electron.
- Internal backend is Express and runs locally.
- UI is dark-mode themed and rendered in the Electron window.
- Mobile mode and desktop mode state switching is implemented as a backend state machine.

Implemented Features (control-app)
1. Desktop shell and navigation
- Sidebar with system tabs and app tabs.
- App tabs are populated from backend app registry.
- Smooth tab traversal and status cards.

2. App registry (WIP modules)
- project-app
- work-app
- calendar-app
- drawing-app
- news-app
- music-app
- server-manager-app
- movie-app

3. Event bus
- In-memory event bus with bounded history.
- API endpoints for listing, posting, and SSE streaming events.
- Dedicated Event Bus tab with fixed-height scroll panel.

4. Mode management
- Backend mode state machine with transition history.
- Header controls to switch Desktop/Mobile mode.

5. Settings tab (desktop-focused)
- Audio device selection.
- Voice selection.
- Voice speed selection.
- Save settings endpoint.
- Test Voice button endpoint (currently local stub behavior + event emission).

6. Chatbot tab
- Text chat available in Desktop Mode.
- Mobile mode shows desktop-required notice.
- Local in-memory chat service and API implemented as integration-ready stub.
- User messages produce local stub agent replies.

7. Startup script
- PowerShell startup script exists in control-app:
  - start-control-center.ps1

How To Run
From D:\Control-Center\control-app:
- Desktop app (preferred):
  - .\start-control-center.ps1
  - or npm start
- Web/backend-only fallback:
  - npm run start:web

- No real voice synthesis/playback pipeline yet (test-voice is stubbed endpoint behavior).
- Most app modules are intentionally WIP shell states with connector stubs only.
