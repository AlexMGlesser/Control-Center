Current App Runtime Model
- Control Center is now a desktop app runtime using Electron.
- Internal backend is Express and runs locally.
- UI is dark-mode themed and rendered in the Electron window.
- Mobile mode and desktop mode state switching is implemented as a backend state machine.
- Agent runtime uses deterministic intent routing first, with local model fallback for non-deterministic prompts.

Implemented Features (control-app)
1. Desktop shell and navigation
- Sidebar with system tabs and app tabs.
- App tabs are populated from backend app registry.
- Smooth tab traversal and status cards.

2. App registry (window-capable modules)

7. Startup script

8. Calendar app (implemented)
- Standalone Calendar app is served from the root workspace `calendar-app` folder.

- Backend service persists events to `control-app/server/data/calendar-events.json`.
- Calendar tool operations implemented:

  - read rest of day

9. Drawing app (implemented)

10. Movie app (implemented)
- Standalone Movie app opens in its own Electron window.
- Folder import, poster-style library cards, inline playback, and playback controls are functional.
- Import overlay correctly hides once movies are loaded.

11. Server Manager app (implemented)

- Server definitions persist locally in browser storage.
- Add Server and Config flows are functional.
- SSH key browsing uses the shared file picker with inline manual path entry fallback.
- SSH launch now routes through a server-side endpoint that opens a PowerShell window directly with the requested SSH command.

12. Voice pipeline (in progress)

Known Limits (by design)
- LLM/tool-call responses can still vary by phrasing when prompts are far from supported deterministic patterns.
- Voice stack binaries/assets are large and should remain intentionally scoped during commits.
- project-app and work-app remain comparatively lighter than the media and server apps.
- Server Manager currently focuses on SSH launch and config management; richer hardware and power telemetry is still pending.
- Movie App currently plays imported local files in-session rather than maintaining a long-lived indexed library.

Recommended Next Priorities
1. Add recurring events and event editing support to the calendar service and UI.
2. Stabilize voice pipeline end-to-end (record, transcribe, route command, speak response).
3. Expand Server Manager with power actions, richer server metadata, and remote storage workflows.
4. Add durable movie library persistence instead of per-session folder import only.
5. Add persistent settings and chat storage (file/db) instead of memory-only runtime state.

Conventions Used
- Local-first desktop execution.
- App isolation via module boundaries and app registry.
- Shared Electron window and dialog plumbing for standalone apps.
- Event-driven status updates via event bus.
- Dark-mode style tokens and consistent UI language.
